Type of contract: Token Migration or Upgrade
Purpose of the contract: Enables token holders to convert their old tokens to new tokens during a migration or upgrade process
Key features: 
1. Token holders can optin to migrate their tokens to a new version of the token contract.
2. The migration process involves burning old tokens and minting new tokens at a 1:1 ratio.
3. The contract maintains the total number of tokens that have been migrated.
4. An upgradeable token can have a new upgradeable token whenever a new version of the contract is created.
5. The contract allows for pausing and resuming the upgrade process as needed.
6. The upgrade process can only be initiated by the contract owner or specific authorized accounts.
