Type of contract: MultiSigWallet
Purpose of the code: To enable multiple owners to jointly control the funds of the contract and to execute transactions only when a predefined number of approvals have been obtained.
Working process: The code allows the addition of multiple owners, and each owner can propose a transaction. The transaction is added to the list of pending transactions. To execute the transaction, a minimum number of owners must confirm (sign) the transaction, after which any owner can execute the confirmed transaction. The executed transaction is marked so that it cannot be executed multiple times.
