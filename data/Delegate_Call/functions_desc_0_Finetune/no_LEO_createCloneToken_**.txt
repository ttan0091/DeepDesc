Type of the Contract: MiniMeToken Factory
Purpose of the Contract: To create MiniMeToken clones with specific settings such as name, symbol, and decimal units, inheriting the snapshot state from a parent token.
Working Process: The developer calls the createCloneToken function on the MiniMeTokenFactory contract, providing the parent token, snapshot block, token name, symbol, decimal units, and transfer status. The function then creates a new MiniMeToken instance by calling the constructor of MiniMeToken.sol, setting the parameters accordingly. After creating the new token, the function returns it to the caller.
