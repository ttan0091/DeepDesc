Type of the script or application: Smart Contract Development
        Main function of the script or application: Track votes for a proposal to determine the votes at the current proposal state, considering actions that can alter voting power like burning or minting.
        Inputs required for the script or application: 
           All proposals of all addresses that have governed tokens, tracked through queue transactions
           Number of unpropose blocks
           Total supply of votes
           List of all proposals with current votes
           The address that burned tokens
           Number of tokens burned
           Minimum number of votes for a proposal to be considered
           Length of the Proposal Queue
           Queue TransactionsmaxLength
           Voting delay and voting period lengths
           Current block number
        Output generated by the script or application: An updated list of proposals and their votes, with additional proposals added due to queue transactions.
